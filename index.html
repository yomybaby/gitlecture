    Cooperation without Coordination

    Permission Culture

    허락은 받은 자만이 사용할 수 있는 문화 : 저작권

    Open Culture

    누구나 자유롭게 이용 할 수 있는 문화 : Creative Commons, Open source

    Github는 기본이 Open

    Github 들여다보기

    Fork

    누구나 소스를 볼 수 있으며 내
    저장소에 fork해서 수정할 수 있다.

    누구나 실제 원본에 영향을 미치지 않고 내 마음대로 수정(개선)할 수 있다.

    Pull request

    누구나 반영을 요청을 할 수 있고
    그것은 함께 공개적으로 논의 된다.

    우리는 실생활에 이런 시스템이 있던가? 요청 절차도 복잡하고 내 요청과 비슷한 요청이 얼마나 있는지 모르며 어떻게 진행되고 있는지 조차 알수 없다.

    diff

    차이점을 쉽게 비교할 수 있다.

    보험약관을 보면 비슷하면서도 다르다. 근데 우리는 다른 차이점을 쉽게 비교할 수 있나?

    link, mention

    모두 연결할 수 있다.

    실제 예

    - fork 해서 pull request 보냄 - diff를 통해 다른 이들이 내 코드에 대해 쉽게 이해함 - 논의의 시작 더 나은 아이디어 - 투표까지 - 결국에는 더나은 새로운 방식으로 다른 담당자에 의해서 구현 - 실제 제품에 반영
